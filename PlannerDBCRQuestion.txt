I am working on a day planner / organizer. It may in the future be combined with a 
project planning tool which is why the `Task` table is so complex. This
database is used in several other questions and previous versions of this 
database have been the basis for several other previous questions:  
 - [Inserting and retrieving data MySql Database access in C++ using Boost::ASIO and Boost::MySQL](https://codereview.stackexchange.com/questions/297855/inserting-and-retrieving-data-mysql-database-access-in-c-using-boostasio-and)  
 - [My first attempt to insert and retrieve data from a MySQL database using the boost::mysql library](https://codereview.stackexchange.com/questions/297856/my-first-attempt-to-insert-and-retrieve-data-from-a-mysql-database-using-the-boo)  
 - [Polymorphic data models that save data to MySQL and restore data from MySQL](https://codereview.stackexchange.com/questions/298180/polymorphic-data-models-that-save-data-to-mysql-and-restore-data-from-mysql)  
 - [Integration Test for Polymorphic data models that save data to MySQL](https://codereview.stackexchange.com/questions/298208/integration-test-for-polymorphic-data-models-that-save-data-to-mysql)  
 - [Self / Unit test of polymorphic database model](https://codereview.stackexchange.com/questions/299494/self-unit-test-of-polymorphic-database-model)

You may notice that every table has a `Hidden` field, this is to allow the application to delete 
records without removing the record from the database.  

The code that uses this database is written in C++. The connection, insertion and deletion
are managed through the `Boost` library `MySQL`. The database server is `MariaDB`. The
application is written using Qt.  

Each table in the database is represented in the C++ code as a Model, for example the `Task` 
table is represented by the `TaskModel` and the `UserScheduleItem` table is represented by
the `ScheduleItemModel`. Each model also has an associated `QueryProcessor` that performs
data retrieval. For example `TaskQueryProcessor` and `ScheduleItemQueryProcessor`. 
Originally the C++ code contained raw `SQL` statements, however, with the introduction 
of the `Hidden` fields I decided to use stored procedures to insert, retrieve and 
delete items instead.  

The code is organized so that all of the relevant stored procedures follow the table 
they modify.  

During testing the database is tested by running the stored procedures in a test `SQL`
file before running the unit and integration tests of the models. There are 2 test
`SQL`, one [creates a test database](https://github.com/pacmaninbw/CCTaskManagementAndScheduler/blob/ScheduleSliceForCodeReview_07052026/Development/Prototypes/DBModelDevAndTest/PlannerTaskScheduleDB.sql`) 
and the [other adds data and runs the stored procedures](https://github.com/pacmaninbw/CCTaskManagementAndScheduler/blob/ScheduleSliceForCodeReview_07052026/Development/Prototypes/DBModelDevAndTest/AdditionalFunctionalTestData.sql).

USE `PlannerTaskScheduleDB`;
DROP function IF EXISTS `PlannerTaskScheduleDB`.`findUserIDKeyByLoginName`;

CREATE FUNCTION `findUserIDKeyByLoginName`(
    LoginName VARCHAR(45)
) RETURNS INT
DETERMINISTIC
BEGIN

    SET @UserIDKey = 0;

    SELECT UserLoginAndPassword.UserID INTO @UserIDKey
        FROM UserLoginAndPassword
        WHERE UserLoginAndPassword.LoginName = LoginName;
    IF @UserIDKey IS NULL THEN
        SET @UserIDKey = 0;
    END IF;

    RETURN @UserIDKey;
    
END$$


-- -----------------------------------------------------
-- function findUserIDKeyByFullName
-- -----------------------------------------------------

DROP function IF EXISTS `PlannerTaskScheduleDB`.`findUserIDKeyByFullName`;

CREATE FUNCTION `findUserIDKeyByFullName`(
    LastName VARCHAR(45),
    FirstName TINYTEXT,
    MiddleInitial TINYTEXT
) RETURNS INT
DETERMINISTIC
BEGIN

    SET @UserIDKey = 0;

    SELECT UserProfile.UserID INTO @UserIDKey
        FROM UserProfile
        WHERE UserProfile.LastName = LastName AND
            UserProfile.FirstName = FirstName AND
            UserProfile.MiddleInitial = MiddleInitial;
    IF @UserIDKey IS NULL THEN
        SET @UserIDKey = 0;
    END IF;

    RETURN @UserIDKey;
    
END$$

-- -----------------------------------------------------

DROP function IF EXISTS `isValidUserLoginAndPassword`;

CREATE FUNCTION `isValidUserLoginAndPassword`
(
    LoginName VARCHAR(45),
    HashedPassWord TINYTEXT
)
RETURNS TINYINT
DETERMINISTIC
BEGIN

    SET @UserIDKey = 0;
    SET @isValid = 1;

    SELECT UserLoginAndPassword.UserID INTO @UserIDKey
        FROM UserLoginAndPassword
        WHERE UserLoginAndPassword.LoginName = LoginName AND
            UserLoginAndPassword.HashedPassWord = HashedPassWord;

    IF @UserIDKey IS NULL THEN
        SET @isValid = 0;
    END IF;

    RETURN @isValid;

END$$

    IN `primaryKeyValue` INT UNSIGNED,
    IN `creatorID` INT UNSIGNED,
    IN `assignedID` INT UNSIGNED,
    IN `description` VARCHAR(256),
    IN `parentTaskID` INT UNSIGNED,

)
